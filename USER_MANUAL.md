## 1. 시스템 소개

### 1.1 돌콩봇이란?

**돌콩봇**은 인공지능 기반의 문서 검색 및 질의응답 시스템입니다. 회사 내부 문서, 업무 매뉴얼, 규정집 등을 업로드하면 돌콩이가 문서 내용을 학습하여 자연어로 질문에 답변해줍니다.

### 1.2 핵심 기술

- **KoSBERT**: 한국어 문장 임베딩 기술로 의미 기반 검색 지원
- **Qdrant**: 고성능 벡터 데이터베이스
- **Google Gemini Pro**: 최신 생성형 AI 언어 모델 (RAG)
- **네이버웍스 OAuth**: 안전한 사용자 인증

### 1.3 시스템 특징

✅ **정확한 검색**: 키워드가 아닌 문맥과 의미를 이해하는 검색  
✅ **자연스러운 대화**: 사람처럼 대화하며 정보를 제공  
✅ **다양한 파일 형식**: PDF, Word, Excel, 텍스트 파일 지원  
✅ **실시간 응답**: 빠르고 정확한 답변 생성  
✅ **보안**: 네이버웍스 인증으로 안전한 접근 제어

1. **웹 브라우저에서 돌콩봇 URL 접속**
   ```
   https://www.yncsmart.com/dollkongbot/
   또는
   http://localhost:3005 (로컬 개발 환경)
   ```

2. **자동 로그인 페이지 리다이렉트**
   - 로그인하지 않은 경우 자동으로 네이버웍스 로그인 페이지로 이동합니다

3. **네이버웍스 로그인**
   - 조직 네이버웍스 계정으로 로그인
   - OAuth 인증 승인

4. **돌콩봇 메인 화면 접속**
   - 로그인 성공 시 자동으로 메인 화면으로 이동

## 4. 기능별 사용법

### 4.1 RAG 채팅 (🤖 RAG 채팅 탭)

업로드된 문서를 기반으로 AI와 대화하며 정보를 얻을 수 있는 기능입니다.

#### 4.1.1 채팅 시작하기

1. **RAG 채팅 탭 선택**
   - 상단 메뉴에서 "🤖 RAG 채팅" 클릭

2. **질문 입력**
   - 화면 하단 입력창에 질문 입력
   - 예시:
     ```
     휴가 신청 방법을 알려줘
     회의실 예약은 어떻게 하나요?
     출장 경비 정산 절차가 궁금합니다
     ```

3. **전송**
   - Enter 키 또는 전송 버튼 클릭
   - 돌콩이가 관련 문서를 찾아 답변 생성

4. **답변 확인**
   - AI가 생성한 답변 확인
   - 출처 문서 정보 확인 (파일명, 유사도 점수)

#### 4.1.2 효과적인 질문 방법

✅ **좋은 질문 예시**
```
❌ "휴가"
✅ "연차 휴가 신청 방법을 알려주세요"

❌ "회의실"
✅ "3층 회의실 예약 방법과 사용 규칙이 궁금합니다"

❌ "급여"
✅ "월급 명세서는 언제 확인할 수 있나요?"
```

#### 4.1.3 채팅 기능

- **💬 질문 및 답변**: 자연스러운 대화형 인터페이스
- **📚 출처 표시**: 답변의 근거가 된 문서 정보 표시
- **⭐ 유사도 점수**: 문서와 질문의 관련도 표시 (0.0 ~ 1.0)
- **🗑️ 대화 초기화**: "대화 초기화" 버튼으로 새로운 대화 시작

#### 4.1.4 FAQ 바로가기

채팅 화면 우측 하단에 자주 묻는 질문(FAQ) 바로가기가 있습니다.

1. **FAQ 아이콘 클릭**
   - 화면 우측 하단의 "💬" 아이콘 클릭

2. **FAQ 목록 확인**
   - 카테고리별 자주 묻는 질문 목록 표시

3. **질문 선택**
   - 원하는 질문 클릭 시 해당 질문이 채팅창에 자동 입력

4. **이메일 문의**
   - FAQ에 없는 내용은 "이메일 문의" 버튼으로 담당자에게 직접 문의

---

### 4.2 문서 검색 (🔍 문서 검색 탭)

업로드된 문서에서 특정 내용을 빠르게 찾을 수 있는 기능입니다.

#### 4.2.1 검색 방법

1. **문서 검색 탭 선택**
   - 상단 메뉴에서 "🔍 문서 검색" 클릭

2. **검색어 입력**
   - 검색창에 찾고자 하는 내용 입력
   - 키워드 또는 문장 모두 가능

3. **검색 실행**
   - Enter 키 또는 검색 버튼 클릭

4. **결과 확인**
   - 검색 결과 목록 확인
   - 각 결과의 유사도 점수 및 파일 정보 표시

#### 4.2.2 검색 결과 화면

```
┌─────────────────────────────────────────────────────┐
│ 📋 검색 결과                  "휴가 신청" 검색 결과  │
├─────────────────────────────────────────────────────┤
│                                                       │
│ 📄 인사규정.pdf                      유사도: 95.3%   │
│ "연차 휴가는 입사일로부터 1년 경과 후 발생하며..."  │
│ 업로드: 2025-10-15                                   │
│                                                       │
├─────────────────────────────────────────────────────┤
│                                                       │
│ 📄 업무매뉴얼.docx                   유사도: 88.7%   │
│ "휴가 신청은 전자결재 시스템을 통해 진행하며..."    │
│ 업로드: 2025-10-10                                   │
│                                                       │
└─────────────────────────────────────────────────────┘
```

#### 4.2.3 검색 팁

- **의미 검색**: 정확한 키워드가 아니어도 유사한 의미의 내용을 찾아줍니다
  ```
  "휴가 신청" → "연차 사용", "휴가 요청" 등도 검색됨
  ```

- **복합 검색**: 여러 단어를 조합하여 검색 가능
  ```
  "회의실 예약 방법"
  "출장 경비 정산 절차"
  ```

- **자연어 질문**: 질문 형태로 검색 가능
  ```
  "어떻게 휴가를 신청하나요?"
  "회의실은 어디서 예약하나요?"
  ```

### 4.3 FAQ 조회

자주 묻는 질문과 답변을 빠르게 확인할 수 있습니다.

#### 4.3.1 FAQ 접근 방법

**방법 1: RAG 채팅 화면에서**
1. RAG 채팅 탭으로 이동
2. 우측 하단 "💬 FAQ" 아이콘 클릭
3. FAQ 목록 팝업 표시

**방법 2: 직접 URL 접근**
```
관리자 페이지 → FAQ 관리 탭에서 조회
```

#### 4.3.2 FAQ 카테고리

FAQ는 카테고리별로 분류되어 있습니다:

- **📋 일반**: 시스템 사용법, 계정 관련
- **💼 인사**: 휴가, 근태, 급여 관련
- **🏢 업무**: 업무 프로세스, 시스템 사용
- **🔧 기술**: 시스템 오류, 기술 지원
- **기타**: 분류되지 않은 질문

#### 4.3.3 FAQ 활용

1. **카테고리 선택**
   - 원하는 카테고리 탭 클릭

2. **질문 검색**
   - 검색창에 키워드 입력으로 FAQ 검색

3. **질문 선택**
   - 질문 클릭 시 답변 확장/축소

4. **채팅에 사용**
   - "채팅에 사용" 버튼 클릭 시 해당 질문이 채팅창에 자동 입력

### 4.4 이메일 문의

FAQ나 채팅으로 해결되지 않는 문의사항을 담당자에게 이메일로 전송할 수 있습니다.

#### 4.4.1 이메일 문의 방법

1. **문의 버튼 클릭**
   - RAG 채팅 화면 우측 하단 FAQ 팝업에서 "이메일 문의" 버튼 클릭

2. **문의 내용 작성**
   ```
   ┌──────────────────────────────────────┐
   │  이메일 문의                          │
   ├──────────────────────────────────────┤
   │  제목: [필수 입력]                    │
   │  ________________________________    │
   │                                       │
   │  내용: [필수 입력]                    │
   │  ________________________________    │
   │  ________________________________    │
   │  ________________________________    │
   │                                       │
   │  [취소]              [전송]          │
   └──────────────────────────────────────┘
   ```

3. **정보 입력**
   - **제목**: 문의 제목 (필수)
   - **내용**: 상세 문의 내용 (필수)
   - 발신자 정보는 로그인 계정에서 자동으로 가져옵니다

4. **전송**
   - "전송" 버튼 클릭
   - 담당자에게 이메일 발송
   - 성공 메시지 확인

#### 4.4.2 효과적인 문의 작성법

✅ **좋은 문의 예시**

```
제목: [시스템 오류] 문서 업로드 실패 문제

내용:
안녕하세요. 홍길동 팀장입니다.

오늘 오후 2시경 문서 업로드를 시도했으나 
다음과 같은 오류가 발생했습니다:

- 파일명: 2025_사업계획서.pdf
- 파일 크기: 8.5MB
- 오류 메시지: "파일 업로드 실패"

해결 방법을 안내해 주시면 감사하겠습니다.
```

## 5. 관리자 기능

⚠️ **관리자 권한이 있는 사용자만 접근 가능합니다.**

### 5.1 관리자 페이지 접근

1. **관리자 탭 선택**

2. **서브 메뉴 선택**
   - **📁 문서 관리**: 문서 업로드 및 삭제
   - **💬 FAQ 관리**: FAQ 등록, 수정, 삭제

---

### 5.2 문서 관리 (📁 문서 관리)

시스템에서 사용할 문서를 업로드하고 관리하는 기능입니다.

#### 5.2.1 문서 업로드

1. **파일 선택**
   - 업로드 영역에 파일 드래그 앤 드롭
   - 또는 "파일 선택" 버튼 클릭

2. **지원 파일 형식**
   - **PDF**: Adobe PDF 파일 (.pdf)
   - **Word**: Microsoft Word 문서 (.docx)
   - **Excel**: Microsoft Excel 파일 (.xlsx)
   - **텍스트**: 텍스트 파일 (.txt)

3. **파일 제한 사항**
   - **최대 크기**: 10MB
   - **파일명**: 한글, 영문, 숫자 사용 권장

4. **업로드 진행**
   ```
   ┌─────────────────────────────────────┐
   │  📁 파일 업로드                     │
   ├─────────────────────────────────────┤
   │                                      │
   │  인사규정.pdf                        │
   │  ████████████░░░░░░░░  65%          │
   │                                      │
   │  처리 중: 텍스트 추출 및 벡터화...  │
   │                                      │
   └─────────────────────────────────────┘
   ```

5. **업로드 완료**
   - 성공 메시지 확인
   - 문서 목록에 자동 추가
   - 즉시 검색 및 채팅에서 사용 가능

#### 5.2.2 업로드 과정

시스템은 다음과 같은 과정으로 문서를 처리합니다:

```
파일 업로드
    ↓
텍스트 추출 (PDF, Word, Excel → 텍스트)
    ↓
형태소 분석 (한국어 문장 분석)
    ↓
청킹 (문장 단위로 분할)
    ↓
벡터 임베딩 (의미를 숫자로 변환)
    ↓
벡터 DB 저장 (검색 가능한 형태로 저장)
    ↓
완료 (검색 및 채팅에서 사용 가능)
```

처리 시간:
- 소규모 문서 (< 1MB): 약 10~30초
- 중규모 문서 (1~5MB): 약 30초~2분
- 대규모 문서 (5~10MB): 약 2~5분

#### 5.2.3 업로드된 문서 목록 확인

```
┌─────────────────────────────────────────────────┐
│ 📚 업로드된 문서                  [새로고침]    │
├─────────────────────────────────────────────────┤
│                                                   │
│ 📄 인사규정.pdf                       [🗑️]      │
│ 123개 청크 • 2025-10-15                          │
│                                                   │
├─────────────────────────────────────────────────┤
│                                                   │
│ 📄 업무매뉴얼.docx                    [🗑️]      │
│ 87개 청크 • 2025-10-10                           │
│                                                   │
└─────────────────────────────────────────────────┘
```

문서 정보:
- **파일명**: 업로드된 파일 이름
- **청크 수**: 문서가 몇 개의 조각으로 나뉘었는지
- **업로드 날짜**: 업로드된 날짜

#### 5.2.4 문서 삭제

1. **삭제 버튼 클릭**
   - 문서 목록에서 삭제할 문서의 🗑️ 아이콘 클릭

2. **확인 메시지**
   ```
   정말로 이 문서를 삭제하시겠습니까?
   
   파일명: 인사규정.pdf
   
   삭제된 문서는 복구할 수 없습니다.
   
   [취소]  [삭제]
   ```

3. **삭제 실행**
   - "삭제" 버튼 클릭
   - 문서 및 관련 벡터 데이터 모두 삭제
   - 검색 및 채팅에서 더 이상 사용되지 않음

⚠️ **주의사항**
- 삭제된 문서는 복구할 수 없습니다
- 삭제 전 반드시 확인하세요
- 중요 문서는 백업 보관을 권장합니다

### 5.3 FAQ 관리 (💬 FAQ 관리)

자주 묻는 질문과 답변을 등록하고 관리하는 기능입니다.

#### 5.3.1 FAQ 등록

1. **"+ FAQ 추가" 버튼 클릭**

2. **FAQ 정보 입력**
   ```
   ┌──────────────────────────────────────┐
   │  FAQ 추가                             │
   ├──────────────────────────────────────┤
   │  카테고리: [선택]     ▼              │
   │    - 일반                             │
   │    - 인사                             │
   │    - 업무                             │
   │    - 기술                             │
   │    - 기타                             │
   │                                       │
   │  질문:                                │
   │  ________________________________    │
   │                                       │
   │  답변:                                │
   │  ________________________________    │
   │  ________________________________    │
   │  ________________________________    │
   │                                       │
   │  키워드: (콤마로 구분)               │
   │  ________________________________    │
   │                                       │
   │  [취소]              [저장]          │
   └──────────────────────────────────────┘
   ```

3. **입력 항목**
   - **카테고리** (필수): FAQ 분류
   - **질문** (필수): 자주 묻는 질문
   - **답변** (필수): 해당 질문에 대한 답변
   - **키워드** (선택): 검색을 위한 키워드 (콤마로 구분)

4. **저장**
   - "저장" 버튼 클릭
   - FAQ 목록에 즉시 추가
   - 사용자에게 즉시 표시됨

#### 5.3.2 FAQ 수정

1. **FAQ 목록에서 수정할 항목 클릭**

2. **"✏️ 수정" 버튼 클릭**

3. **정보 수정**
   - 카테고리, 질문, 답변, 키워드 수정

4. **저장**
   - "저장" 버튼으로 변경사항 저장

#### 5.3.3 FAQ 삭제

1. **FAQ 목록에서 삭제할 항목 선택**

2. **"🗑️ 삭제" 버튼 클릭**

3. **확인**
   ```
   정말로 이 FAQ를 삭제하시겠습니까?
   
   질문: 휴가 신청은 어떻게 하나요?
   
   [취소]  [삭제]
   ```

4. **삭제 실행**
   - FAQ 목록에서 제거
   - 사용자에게 더 이상 표시되지 않음

#### 5.3.4 FAQ 작성 가이드

**좋은 FAQ 예시:**

```
카테고리: 인사
질문: 연차 휴가는 언제 발생하나요?
답변: 연차 휴가는 입사일로부터 1년 경과 후 15일이 발생하며, 
      이후 매년 근속 기간에 따라 1일씩 추가됩니다. 
      최대 25일까지 발생 가능합니다.
키워드: 휴가, 연차, 발생, 휴가일수
```

**작성 팁:**
- 질문은 사용자 관점에서 작성
- 답변은 명확하고 구체적으로
- 관련 키워드를 충분히 입력
- 자주 묻는 순서대로 정렬

## 6. FAQ 및 문제 해결

### 6.1 로그인 관련

#### Q1. 로그인이 되지 않아요

**해결 방법:**
1. 네이버웍스 계정 정보 확인
2. 조직 내 계정인지 확인
3. 네트워크 연결 상태 확인
4. 브라우저 캐시 삭제 후 재시도
5. 관리자에게 계정 권한 확인 요청

#### Q2. 로그인 후 자동으로 로그아웃되어요

**원인:**
- 세션 만료 (일정 시간 미사용 시)
- 토큰 만료

**해결 방법:**
- 다시 로그인
- 브라우저에서 쿠키 차단 해제

---

### 6.2 채팅 관련

#### Q3. 채팅에서 답변이 나오지 않아요

**원인:**
1. 업로드된 문서가 없음
2. 질문과 관련된 문서가 없음
3. 서버 연결 문제

**해결 방법:**
1. 관련 문서가 업로드되어 있는지 확인
2. 질문을 더 구체적으로 작성
3. 잠시 후 다시 시도
4. 관리자에게 문의

#### Q4. 채팅 답변이 부정확해요

**원인:**
- 업로드된 문서에 해당 정보가 없거나 불충분함
- 질문이 모호함

**해결 방법:**
1. 질문을 더 명확하고 구체적으로 작성
2. 키워드를 포함하여 질문
3. 관련 문서가 업로드되어 있는지 확인
4. FAQ에서 유사한 질문 찾기

---

### 6.3 검색 관련

#### Q5. 검색 결과가 나오지 않아요

**원인:**
- 업로드된 문서에 해당 내용이 없음
- 검색어가 너무 구체적임

**해결 방법:**
1. 검색어를 단순화
2. 유사한 의미의 다른 단어 시도
3. 키워드 대신 질문 형태로 검색
4. 관리자에게 관련 문서 업로드 요청

#### Q6. 검색 속도가 느려요

**원인:**
- 서버 부하
- 네트워크 지연
- 대용량 문서 처리 중

**해결 방법:**
1. 잠시 후 다시 시도
2. 네트워크 연결 상태 확인
3. 지속될 경우 관리자에게 문의

---

### 6.4 파일 업로드 관련 (관리자)

#### Q7. 파일 업로드가 실패해요

**원인:**
1. 파일 크기 초과 (10MB 제한)
2. 지원하지 않는 파일 형식
3. 파일 손상
4. 서버 저장 공간 부족

**해결 방법:**
1. 파일 크기 확인 (10MB 이하)
2. 지원 형식 확인 (PDF, DOCX, XLSX, TXT)
3. 파일 정상 여부 확인 (다른 프로그램에서 열기)
4. 파일을 여러 개로 분할하여 업로드
5. 관리자에게 문의

#### Q8. 업로드한 문서가 검색에 안 나와요

**원인:**
- 업로드 처리 중
- 업로드 실패

**해결 방법:**
1. 업로드 완료 메시지 확인
2. 문서 목록에서 파일 확인
3. 잠시 후 다시 검색 (처리 시간 필요)
4. 업로드 재시도

---

### 6.5 이메일 문의 관련

#### Q9. 이메일 문의 전송이 안 돼요

**원인:**
- 필수 항목 미입력
- 서버 오류
- 네트워크 문제

**해결 방법:**
1. 제목과 내용 필수 입력 확인
2. 잠시 후 재시도
3. 관리자에게 직접 연락

---

### 6.6 기타

#### Q10. 화면이 제대로 보이지 않아요

**해결 방법:**
1. 브라우저 새로고침 (F5 또는 Ctrl+R)
2. 브라우저 캐시 삭제
3. 다른 브라우저로 접속
4. 화면 확대/축소 비율 확인 (100% 권장)

#### Q11. 모바일에서 사용할 수 있나요?

**답변:**
네, 모바일 브라우저에서 접속 가능합니다. 
다만 일부 기능은 데스크톱 환경에서 더 편리하게 사용할 수 있습니다.

#### Q12. 시스템이 느려요

**원인:**
- 서버 부하
- 네트워크 지연
- 대용량 문서 처리 중

**해결 방법:**
1. 잠시 후 다시 시도
2. 브라우저 캐시 삭제
3. 다른 탭/창 닫기
4. 관리자에게 문의

## 7. 부록

### 7.1 시스템 아키텍처

```
┌─────────────────────────────────────────────────┐
│              사용자 (웹 브라우저)                │
└─────────────────┬───────────────────────────────┘
                  │ HTTPS
                  ↓
┌─────────────────────────────────────────────────┐
│         프론트엔드 (React + TypeScript)         │
│   - 사용자 인터페이스                            │
│   - 네이버웍스 OAuth 인증                        │
└─────────────────┬───────────────────────────────┘
                  │ REST API
                  ↓
┌─────────────────────────────────────────────────┐
│            백엔드 (FastAPI)                      │
│   ├─ 문서 업로드 처리                            │
│   ├─ 검색 엔진                                   │
│   ├─ RAG 채팅 엔진                               │
│   └─ FAQ 관리                                    │
└────┬────────────────┬──────────────────┬─────────┘
     │                │                  │
     ↓                ↓                  ↓
┌─────────┐  ┌──────────────┐  ┌──────────────┐
│ Qdrant  │  │   KoSBERT    │  │ Gemini Pro   │
│벡터 DB  │  │한국어 임베딩 │  │  LLM 모델    │
└─────────┘  └──────────────┘  └──────────────┘
```

### 7.2 기술 스택 상세

#### 프론트엔드
- **React 18**: 사용자 인터페이스 프레임워크
- **TypeScript**: 정적 타입 언어
- **Tailwind CSS**: 스타일링
- **Vite**: 빌드 도구
- **Axios**: HTTP 클라이언트

#### 백엔드
- **FastAPI**: Python 웹 프레임워크
- **KoSBERT**: 한국어 문장 임베딩 (jhgan/ko-sbert-nli)
- **Kiwi (kiwipiepy)**: 한국어 형태소 분석기
- **Qdrant**: 벡터 데이터베이스
- **Google Gemini Pro**: 생성형 AI 모델

#### 파일 처리
- **PyPDF2**: PDF 파일 파싱
- **python-docx**: Word 문서 파싱
- **openpyxl**: Excel 파일 파싱

### 7.3 용어 설명

| 용어 | 설명 |
|------|------|
| **RAG** | Retrieval-Augmented Generation. 문서 검색 결과를 바탕으로 AI가 답변을 생성하는 기술 |
| **임베딩** | 텍스트를 숫자 벡터로 변환하는 과정. 의미를 수치화함 |
| **벡터 DB** | 벡터(숫자 배열) 형태의 데이터를 효율적으로 저장하고 검색하는 데이터베이스 |
| **청킹** | 긴 문서를 작은 단위(문장, 단락)로 나누는 과정 |
| **코사인 유사도** | 두 벡터 간의 유사도를 측정하는 방법. 0~1 사이 값 (1에 가까울수록 유사) |
| **OAuth** | 안전한 인증을 위한 표준 프로토콜 |
| **토큰** | 인증된 사용자임을 증명하는 암호화된 문자열 |

### 7.4 성능 지표

| 항목 | 성능 |
|------|------|
| **검색 속도** | 평균 100~300ms |
| **채팅 응답 속도** | 평균 2~5초 |
| **문서 업로드 처리** | 1MB당 약 10~30초 |
| **최대 파일 크기** | 10MB |
| **동시 사용자** | 최대 100명 (서버 사양에 따라 다름) |
| **임베딩 차원** | 768차원 |
| **유사도 임계값** | 기본 0.7 (조정 가능) |

### 7.5 보안 및 개인정보 보호

- **인증**: 네이버웍스 OAuth 2.0 기반 안전한 인증

- **로그 기록**: 모든 작업 기록 저장 (감사 추적)
